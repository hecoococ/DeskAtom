# 阿里云实时语音识别 Electron 集成计划

## 问题分析

测试页面（`voice-test.html`）正常工作，但 Electron 应用中点击录音按钮后窗口消失。

**根本原因**：Electron 透明窗口（`transparent: true`）+ `alwaysOnTop: true` 组合下，音频处理链连接 `destination` 会触发窗口行为异常。

## 解决方案

参考测试页面成功的实现，使用 **`gainNode.gain.value = 0`** 静音连接到 destination，既能触发 `onaudioprocess` 事件，又不会播放声音。

## 实施步骤

### 步骤 1：修改 VoiceInput.vue
**文件**：`src/components/VoiceInput.vue`

按照测试页面的方式重写音频处理：

```javascript
// 启动麦克风采集
async function startMicrophoneCapture() {
  // 1. 获取麦克风权限
  mediaStream = await navigator.mediaDevices.getUserMedia({
    audio: { sampleRate: 16000, channelCount: 1, ... }
  })

  // 2. 创建 AudioContext
  audioContext = new AudioContext({ sampleRate: 16000 })
  const source = audioContext.createMediaStreamSource(mediaStream)
  const processor = audioContext.createScriptProcessor(4096, 1, 1)

  // 3. 关键：静音 gainNode 连接到 destination
  const gainNode = audioContext.createGain()
  gainNode.gain.value = 0  // 静音，不播放声音

  // 4. 连接链：麦克风 → source → processor → gainNode → destination
  source.connect(processor)
  processor.connect(gainNode)
  gainNode.connect(audioContext.destination)

  // 5. 处理音频数据
  processor.onaudioprocess = (event) => {
    if (window.electronAPI && window.audioRecordingReady) {
      const inputData = event.inputBuffer.getChannelData(0)
      const int16Array = convertFloat32ToInt16(inputData)
      window.electronAPI.voiceSendAudio(Array.from(int16Array))
    }
  }
}
```
